onnx

发表于 2022-04-14 更新于 2024-07-30 Valine：
本文字数： 3.4k 阅读时长 ≈ 3 分钟

base

  1. 打印信息
  1. wget https://github.com/onnx/models/raw/b9a54e89508f101a1611cd64f4ef56b9cb62c7cf/vision/classification/resnet/model/resnet50-v2-7.onnx
  2. resize 算子示例
    • Attributes(注意和输入区别)
    • Inputs
    • Outputs
    • Examples
  3. model = onnx.load("mnist_cc.onnx") 模型无结构信息，print(model)无结果，model 是 onnx.onnx_ml_pb2.ModelProto 类型
  4. vscode 使用 vscode-netron 可以和源码一起对比看
  5. onnx: 奥尼克死
  6. vscode 打开 onnx 文件，可以看出 op 生成所依赖的文件路径
  7. onnx 支持设置明确 batch size. By default, TensorFlow does not set an explicit batch size. link
  8. why onnx?
  9. ONNX （Open Neural Network Exchange）
  10. ONNX 文件不仅仅存储了神经网络模型的权重，同时也存储了模型的结构信息以及网络中每一层的输入输出和一些其它的辅助信息。
  11. ONNX 使用的是 Protobuf 这个序列化数据结构去存储神经网络的权重信息。
  12. onnx 实现了一个 python 运行时，可用于评估 ONNX 模型和评估 ONNX 操作。目的是阐明 ONNX 的语义并帮助理解和调试 ONNX 工具和转换器。它无意用于生产，性能也不是目标
  13. ONNX 是强类型的，其定义不支持隐式强制转换。
  14. Constant 值预先已知，用于存 weights 和 bias,
  15. Variable 值在推理时之前未知，但可能包含有关数据类型和形状的信息。用于存中间结果
  16. Constant 和 Variable 都是 tensor
  17. 在 ONNX (Open Neural Network Exchange) 模型中，”attributes” 是与模型的各个节点或算子相关联的关键信息。这些属性为算子提供了额外的规范和配置信息，使得算子能够正确执行其预定的操作。理解属性在 ONNX 模型中的作用是重要的，尤其是在模型转换、优化或理解模型行为时。
  18. 属性用于指定算子的具体行为。例如，在一个卷积算子中，属性可以定义卷积核的大小、步长（stride）、填充（padding）等。
  19. 不变参数(weights bias div mul 需要的参数)使用 constant tensor 类型，输入是 variable tensor, attributes 是 op 初始化用到的数值， 计算不用
  20. onnx 模型 variable tensor 不需要 type 信息，转化后才有信息; weights 有 type 信息

netron

  1. 神经网络本质上是一个计算图。计算图的节点是算子，边是参与运算的张量
  2. 线是 tensor, 方块是 op，op 中的 weight 也是 tensor, op 中的 type 是算子类型，输入 tensor 和 op weight 根据算子类型和参数计算得到输出 tensor
  3. op 可以想象成函数，tensor 和属性是函数输入输出参数
  4. samples 直接看
  5. 可以显示 type shape 等信息
  6. 显示 shape 信息（需要实例）
  1. netron 点击 input 或 output 可以显示模型整体信息
  2. netron 查看 weights 需要点击+
  3. netron 左侧菜单栏可以点击 toggle names, weights, attributes
  4. Netron supports ONNX, TensorFlow Lite, Core ML, Keras, Caffe, Darknet, MXNet, PaddlePaddle, ncnn, MNN and TensorFlow.js. Netron has experimental support for PyTorch, TorchScript, TensorFlow, OpenVINO, RKNN, MediaPipe, ML.NET and scikit-learn.
