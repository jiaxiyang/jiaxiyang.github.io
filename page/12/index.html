调度框架实现

  1. queue 示意图

心得

  1. 顶点是函数，边是数据
  2. 所谓 flow 是指计算顺序，数据并不会移动。
  3. 顶点和边组成 DAG
  4. DAG 需要调度器来运行
  5. 各框架：
    • vitis ai runtime: op 是顶点，tensor 是边, runner 是调度器
    • dpx flow: module(forward?) 是顶点，message 是边, engine 是调度器
    • taskflow: Node 是顶点，Executor 是调度器
    • CGraph:
    • sogou workflow:
    • tensorflow:
    • pytorch:
  6. CGraph 中的claster和 flow 中的 module 很像, 继承自 group，由多个 functionNode 线性组合而成。执行 cluster 的时候，内部的 node 依次顺序执行。简而言之就是可以依次完成多个功能。
  7. flow 是异步事件驱动的运行模式，使用异步回调方式，feed msg 后执行完 workflow 后调用回调函数。
  8. taskflow 也是异步模式，都有 observer, 执行完后调用回调函数，观察者模式。 tf::ObserverInterface 和 RunObserver
    q1. 顶点需要注册到图中, 并设定依赖关系。

onnx

发表于 2022-04-14 更新于 2024-01-05 Valine：
本文字数： 1.4k 阅读时长 ≈ 1 分钟

base

  1. onnx: 奥尼克死
  2. vscode 打开 onnx 文件，可以看出 op 生成所依赖的文件路径
  3. onnx 支持设置明确 batch size. By default, TensorFlow does not set an explicit batch size. link
  4. why onnx?
  5. ONNX （Open Neural Network Exchange）
  6. ONNX 文件不仅仅存储了神经网络模型的权重，同时也存储了模型的结构信息以及网络中每一层的输入输出和一些其它的辅助信息。
  7. ONNX 使用的是 Protobuf 这个序列化数据结构去存储神经网络的权重信息。
  8. onnx 实现了一个 python 运行时，可用于评估 ONNX 模型和评估 ONNX 操作。目的是阐明 ONNX 的语义并帮助理解和调试 ONNX 工具和转换器。它无意用于生产，性能也不是目标
  9. ONNX 是强类型的，其定义不支持隐式强制转换。
