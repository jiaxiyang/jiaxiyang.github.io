Why Rust

本文字数： 798 阅读时长 ≈ 1 分钟

Why Rust?

Performance

  1. It’s fast and memory efficient: with no runtime or garbage collector.
  2. It can easily integrate with other languages.

Reliability

  1. It’s rich type system and ownership model guarantee memory-safety and thread-safety.

Productivity

  1. Great documentation
  2. Friendly compiler, find most errors when compiling
  3. An integrated package manager
  4. Doc system: cargo doc --open

Rust features

  1. Zero-cost abstractions
  2. Modern conveniences
  3. Type system: Ownership and Borrowing
  4. Sense of craftsmanship

Why Name Rust?

  1. It’s a pun on “chrome”. Rust was written by the engineer company Mozilla(firefox). And chrome is a metal.

Rust 目的

  1. 创建这个新语言的目的是为了解决一个顽疾：软件的演进速度大大低于硬件的演进，软件在语言级别上无法真正利用多核计算带来的性能提升。Rust是针对多核体系提出的语言，并且吸收一些其他动态语言的重要特性，比如不需要管理内存，比如不会出现Null指针等

Why not C++

  1. 当然C++ 也很好，因为它教会了我怎么面向搜索引擎编程。

Rust类型系统借鉴Haskell特性

  1. 没有空指针
  2. 类型默认不可变
  3. 表达式
  4. 高阶函数
  5. 代数数据类型
  6. 模式匹配
  7. 泛型
  8. trait和关联类型
  9. 本地类型推导

Rust相比Haskell独有特点

  1. 仿射变换(Affine Type)
  2. 借用，生命周期

Notes

  1. 不同版本的 clang-format 输出格式可能不同，需要统一团队版本
  2. 文件中有编码错误的字符不能被 format

format check

  1. run-clang-format
    • -r samples 文件夹
    • -i 修改
  2. github clang-format-check
  3. enforce-code-consistency-clang-format#enforce_the_new_coding_rules
  4. clang-format --Werror -n --verbose test.cpp 可以检测是否符合 clang-format
  5. cpplint
  6. gitlab check
  7. pre-commit
  8. git pre commit

Install clang

  1. Build clang from source
  2. Clang format doc

format config

  1. Clang format style
  2. clang format config 试验
  3. AllowShortFunctionsOnASingleLine: None 函数不在一行
  4. SortIncludes: false 头文件不排序

Bash Command

  1. generate .clang-format
    clang-format -style=llvm -dump-config > .clang-format
  2. Format one cpp file:
    clang-format --style=Google -i test.cpp
  3. Format all cpp files:
    fd ".cpp$" | xargs clang-format --style=Google -i

git

  1. git-clang-format-12 or git clang-format-12 format staged files
  2. git-clang-format-12 --diff 查看要 format 的变化

Vim Config

  1. Find clang-format.py in your system and copy the file path.
  1. Put the following config in your .vimrc. It will automaticly format the c++ file when the file is saved.

Emacs Config

  1. use package config

Config

Clang-format understands also special comments that switch formatting in a delimited range. The code between a comment // clang-format off or /* clang-format off */ up to a comment // clang-format on or /* clang-format on */ will not be formatted. The comments themselves will be formatted (aligned) normally.
