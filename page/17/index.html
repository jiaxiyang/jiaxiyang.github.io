Debug

发表于 2022-02-03 更新于 2023-09-24 分类于 Program ， Cpp Valine：
本文字数： 16k 阅读时长 ≈ 14 分钟

NOTE

  1. gcc/g++ --verbose显示编译过程
  2. strip 是一个在 Unix-like 操作系统中用于删除或剥离目标文件或对象文件的符号信息的工具。
    • 删除符号表信息：符号表包含了源代码中的函数和变量的名称。
    • 删除调试信息：debug section 被删除
  3. function tracing 只需要 symbols，不需要 debug symbols（gcc -g）。strip 之后不能跟踪
  4. 动态符号（.dynsym）vs. 局部符号（.symtab）， .symtab 包含.dynsym; 可以用命令 strip 来手动去掉局部符号表， strip 前后用readelf -s ./test看, strip 只去除了.symtab
  5. strip 之后 gdb 不能使用 function 名字加断点
  6. debug info 作用：
    • 将内存地址映射到具体某行源代码
    • 调用栈展开（stack unwinding）
  7. 在 C++中，如果函数有指定返回类型，但在实际代码中没有返回值或者在所有路径上都没有返回值，那么它的行为是未定义的（Undefined Behavior）。根据 C++标准，未定义的行为可能导致任何结果，包括程序崩溃、生成错误的结果或者在某些情况下看起来像是“正常工作”。
  8. addr2line其功能是将函数地址解析成文件名或行号的工具。 addr2line debug
  9. 查看 debug 模式：gdb l and info source
  10. -DCMAKE_CXX_FLAGS="-g" cmake 编译选项添加 debug info； release 模式也适用
  11. gcc 选项顺序很重要，如果两个选项冲突，则以后一个为准。可以用-O3 -fno-inline-functions既使用-O3 的功能又关闭函数内嵌功能。
