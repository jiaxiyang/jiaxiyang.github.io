    Programming experience

    1. 最重要的是官方资源，看英文原文
    2. 文档越详细越好，肯定会忘
    3. Github awesome系列，awesome emacs, shell，rust
    4. git 管理代码，配置，日志
    5. 统一编辑器，emacs
    6. emacs org管理项目，todo，日志
    7. 遇见问题记得看github issues
    8. 设计的思路，设计原则，理解之后对使用帮助很大(例如： emacs, git)
    9. 编译时出错仔细看错误信息
    10. 调试时要一小步一小步的调，步步为营

    Productivity

    1. 用好tmux, fd, rg, zh, fzf
    2. 使用格式化工具刷代码，不要手动刷
    3. 开始时多使用cheatsheet
    4. 常用命令alias

    Keyboard usage

    1. EMACS alt, shift一只手同时按，另一只手按%,>
    2. sharpkeys修改键位
    3. emac快捷键设置技巧
      • 右手leader key,左手按键
      • 常用命令才设置，不常用直接输命令
      • 快捷键有意义
      • 尽量简单简洁
      • 尽量少用左手小拇指，多用命令
