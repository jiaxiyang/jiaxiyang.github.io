    Programming experience

    1. 最重要的是官方资源，看英文原文，如rust官网教程，c++ cppcon会议。
    2. 先见识，再模仿，后摸索。学习新语言，可以先看看视频教程，再自己摸索。
    3. 无论是整体还是模块，都要先学习基本概念, 然后学习框架和设计模式，最后再学习具体代码。
    4. 概念刚开始可能很难理解，可以从具体实例来理解。
    5. 学习建模，对系统建模。
    6. 读代码时要先理解总体设计架构，设计思路，再具体到细节，架构理解了代码就不是问题。
    7. 多学点架构，设计方法，少学点语法。
    8. 架构是撑起项目的基础，理解架构了就能看清楚系统轮廓。架构就相当于系统的草图，有了草图才能起高楼。
    9. 注意软件架构和设计模式的区别。
    10. 设计是解决问题的方案，不是算法。
    11. 看文档或代码的时候抓关键点记笔记，有利于专注，记录的时候会强化，加深理解，可以记录到blog或者git上。如有需要进行可视化
    12. 有目标或者带着问题看文档或代码。
    13. 分主题分模块学习语言，分拆效率高，更容易focus，不能一股脑的啥都往脑子里塞。
    14. 学习新知识时先明确概念，如cy学习git，明确git的原理，四个object；学习cmake时明确component和target；学习gstreamer明确component,pad等，给别人讲解是也是从基本概念、基本组成讲起。
    15. 文档越详细越好，肯定会忘
    16. 读代码的时候可以记录自己的理解，可加深印象。用git仓库保存起来。
    17. github排名，github trending
    18. 查看开发者的github找资料
    19. 先模仿github开源代码。youtube视频教程。
    20. 随时写一些简单的例子验证想法。很重要。
    21. 找最专业的，从原理上学习。如rust设计者讲rust，git原理。
    22. Github awesome系列，awesome emacs, shell，rust
    23. emacs org管理项目，todo，日志
    24. 遇见问题记得看github issues
    25. 设计的思路，设计原则，理解之后对使用帮助很大(例如： emacs, git)
    26. 编译时出错仔细看错误信息
    27. 调试时要一小步一小步的调，步步为营
    28. 乐于学习新东西，学习emacs之后才知道vim该怎么用，学习Rust之后才知道语言一些基本特性，更好的理解C++
    29. 重头文件(接口)，轻源文件。看一个新库时例如xir, vart，先看接口（接口注释很重要），知道接口功能就行，不需要明白具体实现。通过接口明白整个库的结构。
    30. 接口注释相当于leetcode刷题时的思路，.cpp相当于思路的具体实现。具体实现根本记不得，但接口或思路看一下就知道功能或实现方法
    31. 突出重点:读音加重，停顿。看C++代码要具备忽略次要信息的能力，看重点的信息要用不同的眼光，例如函数名，函数功能重点看和理解，具体实现可以先不用管
    32. 程序有Debug和Release模式，Release开优化，编译慢，运行快
    33. 程序有编译期和运行期。
    34. 程序包含功能和性能，注重分析程序的性能，比如帧率，时间
    35. 流程: 开发(开发新库），测试(jenkins)，发布(上github)，运营(gihub issues)
    36. 重设计，轻实现
    37. 按部就班，不能急于求成
    38. 知乎话题：rust, emacs
    39. 学习编程语言要关注几点：起因，解决什么问题，设计哲学，特性 参考文章
    40. 大项目中，模块之间要检查输入输出是否正确，不用在乎模块里内部实现
    41. 日志写在实现的外面，可以从使用者的角度检查输出，日志写在里面，可以从实现者的角度检查问题。倾向写在实现外面，先定位哪个模块出问题了，再深入到模块内部，看看为什么出问题。
    42. 一个函数完成一个小功能，方便理解定位问题
    43. main.rs函数中处理程序运行，lib.rs中处理任务逻辑
    44. 操作符重载打印结构体很方便
    45. API文档目的是让他人理解如何使用库，而不是让他人明白库如何实现的。
    46. 创建一个有用的公有 API 结构更像是一门艺术而非科学，你可以反复检视他们来找出最适合用户的 API
    47. 留意FAQ
    48. 函数(一层抽象)，类，泛型都是为了重复代码。
    49. 写代码时明确（输入，输出，算法）（开，关，做）（构造，析构，虚函数)
    50. RFC流程。
    51. C++ API -> C API 可能拆分， C API -> Rust API 可能合并
    52. 区分公开与非公开部分。

    Productivity

    1. 用好tmux, fd, rg, zh, fzf
    2. 使用格式化工具刷代码，不要手动刷
    3. 开始时多使用cheatsheet
    4. 常用命令alias
    5. 保持工作空间整洁，经常清理，立即删除无用文件
    6. git 管理代码，配置，日志
    7. 多用google，少用百度
    8. 统一编辑器，emacs

    Keyboard usage

    1. EMACS alt, shift一只手同时按，另一只手按%,>
    2. sharpkeys修改键位
    3. emac快捷键设置技巧
      • 右手leader key,左手按键
      • 常用命令才设置，不常用直接输命令
      • 快捷键有意义
      • 尽量简单简洁
      • 尽量少用左手小拇指，多用命令
