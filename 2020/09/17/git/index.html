Worktree

Git worktree 允许同时检出多个分支到不同目录，无需多次 clone。

使用场景

场景 说明
并行开发 同时在多个分支工作，无需 stash 或 commit
紧急修复 开发中途快速切换到 hotfix 分支
代码对比 在不同目录查看不同分支代码
长时间编译 一个 worktree 编译，另一个继续开发
CI/CD 并行构建多个分支

示例：并行开发

注意事项

  • 每个分支只能在一个 worktree 中检出
  • 所有 worktree 共享 .git 目录（在主 worktree）
  • 删除 worktree 前确保没有未提交的更改

原理

Git 是内容寻址文件系统，核心是键值对数据库。

.git 目录结构

目录/文件 说明
objects/ 存储所有数据内容（blob、tree、commit）
refs/ 存储分支、远程、标签的引用指针
HEAD 指向当前检出的分支
index 暂存区信息
config 项目配置
hooks/ 钩子脚本

三种对象类型

类型 说明
blob（数据对象） 文件内容
tree（树对象） 目录结构，包含 blob 和子 tree 的引用
commit（提交对象） 指向 tree，包含作者、时间、父提交信息

底层命令

引用（refs）

  • 分支：指向某个 commit 的指针，存储在 .git/refs/heads/
  • HEAD：符号引用，指向当前分支

核心流程

git add + git commit 实际做了：

  1. 将文件内容保存为 blob 对象
  2. 更新暂存区（index）
  3. 创建 tree 对象
  4. 创建 commit 对象（指向 tree + 父 commit）
