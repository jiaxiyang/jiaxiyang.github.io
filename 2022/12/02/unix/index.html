UNIX 环境高级编程

  1. ID

    • 文件(文件描述符)
    • 进程
    • 错误处理(errno)
    • 用户标识(用户 ID, 组 ID)
    • 信号(SIG)

  2. stat: 获取文件各种信息

    • stat -c %s file 查看文件长度

  3. strace - trace system calls and signals

文件

IO

信号

  1. 信号是软件中断。
  2. 终端键盘上有两种信号方法：
    • SIGINT 中断键：C-c
    • SIGQUIT 退出键：C-\
  3. 使用 system 而不是直接用 fork 和 exec 的优点是： system 有各种错误处理和各种信号处理

进程

  1. 进程 ID 用 pid_t 数据类型来表示，是一个非负整数。 线程 ID 是用 pthread_t 数据类型来表示， 是一个结构体， 不同系统类型不同。
  2. 如果进程中的任意线程调用了 exit, _Exit 或者_exit, 那么整个进程就会终止。
  3. nm -C test | grep GLIBC_可以看出 c++程序用 thread 和 mutex 最终调用了 pthread_create 和 pthread_mutex*
  4. 守护进程出错记录使用 syslog，log 位置 /var/log/syslog

进程间通信 InterProcess Communication IPC

  1. 同一机器：管道，FIFO, 消息队列，信号量，共享内存。
  2. 不同机器：socket

线程间通信

  1. mutex, 读写锁，自旋锁， 条件变量
  2. mutex 保护的是 lock 和 unlock 之间的代码
  3. 如果线程对一个互斥量加锁两次，那么它自身就会陷入死锁。
  4. 线程间循环依赖锁也会死锁，解决办法：1. 控制互斥量加锁的顺序避免死锁。2. try_lock 如果获得不了想要的锁， 先释放已占有的锁，过一段时间再试试。
  5. 自旋锁与互斥量类似， 但自旋锁获取锁之前线程是非阻塞状态。适用于锁持有时间段，线程不希望在重新调度上花费太多成本。自旋锁在非抢占式内核中非常有用，在用户层，自旋锁并不是很有用。
  6. 很多互斥量实现非常高效，有些互斥量的实现在试图获取互斥量的时候会自选一小段时间，只有自旋计数达到某一阈值的时候才会休眠。
