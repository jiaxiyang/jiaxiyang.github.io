Note

  1. 核函数 K（kernel function）就是指 K(x, y) = ，其中 x 和 y 是 n 维的输入值，f(·) 是从 n 维到 m 维的映射（通常而言，m>>n）。是 x 和 y 的内积（inner product），严格来说 应该叫欧式空间的标准内积，也就是很多人常说的点积（dot product）。
  2. sudo update-alternatives --display cuda显示系统 cuda 版本
  3. export PATH=/usr/local/cuda/bin/:${PATH}找不到 nvcc 可能需要 export PATH
  4. sudo apt install -y nvidia-cuda-toolkit 安装 cuda 工具链, 不能乱用，需要和系统 cuda 匹配
  5. 在 CUDA 中，你会以类似于 C/C++函数的形式来表达想要在 GPU 上运行的计算，这个函数被称为 kernel。
  6. CUDA 流表示一个 GPU 操作队列，该队列中的操作将以添加到流中的先后顺序而依次执行。
  7. stream 作用：在 Stream 的帮助下，CUDA 程序可以有效地将内存读取和数值运算并行，从而提升数据的吞吐量。 link
    由于 GPU 和 CPU 不能直接读取对方的内存，CUDA 程序一般会有一下三个步骤：1）将数据从 CPU 内存转移到 GPU 内存，2）GPU 进行运算并将结果保存在 GPU 内存，3）将结果从 GPU 内存拷贝到 CPU 内存。
  8. cuda7 可以开启每个线程有一个默认 stream, 之前每个设备有一个 stream gpu-pro-tip-cuda-7-streams-simplify-concurrency/
  9. GPU 函数耗时统计不能只记录一次的，GPU 可能做一些准备工作，教训： nppiResize_8u_C3R 不管大小第一次运行耗时都很大 The cuda context is lazily initialized
  10. unified memory cpu 访问时需要同一个线程。不同线程会 bus error。
  11. autotuning 搜索 kernel grid 划分参数(结果不变)，找性能最优
  12. L1 和 shared memory 共享一块存储 可动态分配比例; 可用比例见link
  13. warp 调度:延时隐藏 只执行 warp 一部分 当 warp 需要等待时 先执行其他 warp
  14. warp 执行类似 simd
  15. FMA (Fused Multiply Add): z=a*x+y …z,x,y are vectors or scalars
  16. 4FMA (Quad FMA): z=A*x+z …A is a FP32 matrix; x,z are vectors
  17. WMMA: Warp-level Matrix Mulitply and Accumulate (Tensor Core): Z=AB+C …A,B are FP16 matrices; Z,C are FP32

tenosr core 和 cuda core

  1. CUDA Core 是为通用计算设计，而 Tensor Core 是为特定类型的计算（主要是深度学习中的矩阵运算）优化。
  2. 在 NVIDIA 的某些 GPU 架构中，例如 Volta、Turing 和 Ampere，CUDA Core 和 Tensor Core 共同存在。它们可以根据计算任务的性质协同工作，提高整体的计算效率。
  3. 在执行深度学习任务时，Tensor Core 可以显著加速计算过程，相较于仅使用 CUDA Core，能实现更快的训练和推理速度。
  4. RT core 用于光线追踪
  5. 芯片手册中有 cuda core 算力和 tensor core 算力
  6. 利用 tensor core 才能达到最大算力
