Artifactory

  1. 上传最好不要用 sudo 用户，否则容易覆盖之前文件，不好管理
  2. guest 用户可以禁止 ui 登录，防止用户被锁
  3. 可以通过界面上传文件
  4. 作为一个二进制包管理工具，可以用于管理构建产物：即把生成的 jar 包存放到 JFrog，当在生产环境进行部署时，Jenkins 直接从 JFrog 中拉取对应的 jar 包即可。
  5. 官方的介绍：JFrog Artifactory 是一个可扩展的通用二进制存储库管理器，可在整个应用程序开发和交付过程中自动管理你的工件和依赖项。一句话概括：Artifactory 是一个存放制品（Artifacts）的工具。
  6. 左下角 getting started
  7. 允许匿名用户，否则 wget curl 下载不下来， link
  8. 先创建 repo，再上传
  9. repo 分为：local, remote 和 virtual
  10. monitoring -> storage 可以看各 repo 存储大小
  11. Application -> Artifactory -> artifacts 右上角 Set me up 可以查看 deploy 和 resolve 命令
  12. set me up 中可以设置 configue 来隐藏密码
  13. artifacts 中的文件可以单独下载，右上角下载符号
  14. 有些文件可以预览，右上角…展开, 或者选中文件，右击
  15. 可以查看下载次数和被谁下载
  16. artifacts 布局 layouts 可以查看版本, 可以生成各种工具依赖的版本信息
  17. MD5 check：echo 99 > test.txt && MD5=$(md5sum test.txt | awk '{print $1}') && curl -uxiyang.jia:APAQR97hg1swsSNZ -T test.txt -H "X-Checksum-MD5:${MD5}" -L "http://artifacts.phigent.io/artifactory/xiyang-test/test/test/test-1.0.3.txt" -v
  18. 可以通过 name, checksum, property 搜索 artifacts
  19. wget -O test.tar.gz --no-check-certificate https://guest:Guest.123@xxx/artifactory/test.tar.gz 下载, 覆盖原来文件
  20. curl -sSfLO https://guest:Guest.123@xxx/artifactory/test.tar.gz 下载
