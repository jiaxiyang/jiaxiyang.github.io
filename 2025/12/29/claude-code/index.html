CLAUDE.md

什么是 CLAUDE.md

CLAUDE.md 是项目级配置文件，为 Claude Code 提供工程上下文。它指导 Claude 理解项目结构、命令、规范等。

创建与位置

IDE 集成

CLI 和 IDE 插件连接：

  • 在 VS Code 集成终端运行 claude 会自动与 IDE 集成
  • 外部终端使用 /ide 命令连接到 VS Code
  • tmux 首次使用需全部关闭重启
  • 确保 IDE CLI 可用：VS Code 需要 code 命令，Cursor 需要 cursor 命令
  • 参考：https://code.claude.com/docs/en/vs-code#standalone-cli-not-connecting-to-ide
  • claude code show logs
  • reset location来刷新自定义command
    VS Code 快捷键：View: Focus into Secondary Side Bar 绑定到 C-2

工作流

推荐流程：

  • Explore → Plan → Code → Commit
  • Write tests → Commit → Code → Iterate → Commit
  • Write code → Screenshot result → Iterate

优化技巧：

  • 指令要具体明确
  • 提供截图给 Claude
  • 大型任务使用 Markdown 文件作为 checklist
  • 多 Claude 协作：一个写代码，另一个验证；或一个写测试，另一个实现

Skill（技能）

什么是 Skill

Skill 是 Markdown 文件，教导 Claude 如何执行特定任务。与 Slash Command 不同，Skill 由 Claude 自动触发（基于请求内容语义匹配）。

Plugin（插件）

什么是 Plugin

Plugin 是 Claude Code 的扩展包，可包含：

  • 斜杠命令（Commands）
  • 技能（Skills）
  • 事件钩子（Hooks）
  • MCP 服务器
  • LSP 服务器

用途：跨项目复用、团队共享、通过 Marketplace 分发

Plugin vs 独立配置

方式 命令格式 适用场景
独立配置 .claude/ /hello 个人、项目特定、快速实验
Plugin /plugin-name:hello 团队共享、跨项目、版本管理

Boris 的 13 条建议

Claude Code 创始人 Boris 的使用技巧：

  1. 并行多 Claude - 终端运行 5 个 Claude，标签编号 1-5，配合系统通知
  2. 本地 + Web - 终端与 claude.ai/code 并行，用 & 交本地会话给 Web，或 --teleport 来回切换
  3. Opus 4.5 + Thinking - 最佳编码模型，虽大且慢但需引导少，总体更快
  4. 共享 CLAUDE.md - 团队共用一个 CLAUDE.md��检入 git，发现问题立即添加
  5. PR 审查集成 - 用 GitHub Action，在同事 PR 上 tag @.claude 要求更新 CLAUDE.md
  6. Plan 模式优先 - 按 Shift+Tab 两次进入 Plan 模式，计划好后切换 auto-accept 模式一气呵成
  7. Slash 命令 - 每日重复的工作流用命令自动化，如 /commit-push-pr
  8. Subagent - 常用工作流用 subagent，如 code-simplifier、verify-app
  9. PostToolUse Hook - 格式化代码，处理最后 10%
  10. 权限管理 - 不用 --dangerously-skip-permissions，用 /permissions 预授权安全命令
  11. MCP 工具 - Claude 代用所有工具：Slack、BigQuery、Sentry 等
  12. 长时间任务 - 用 background agent 验证，或用 Stop hook，或 ralph-wiggum 插件
  13. 验证反馈循环 - 最重要 - 给 Claude 验证工作的方式，质量提升 2-3 倍
