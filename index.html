Debug

发表于 2022-02-03 更新于 2024-01-18 分类于 Program ， Cpp Valine：
本文字数： 17k 阅读时长 ≈ 15 分钟

NOTE

  1. 没 link 报undefined reference to, link 但没 found xxx was not found, cannot find xxx
  2. target_link_options(xxx PUBLIC -Wl,-unresolved-symbols=ignore-in-shared-libs) 只要能找到 xxx 中的符号就行， link 的 shared lib 中未找到符号不报错， 忽略 link 的库解析，用于交叉编译，运行时能找到库
  3. c++23 支持 stack trace basic_stacktrace
    • backward-cpp
    • cpptrace
    • boost stack trace
  4. 二进制依赖时， 需要先 find_package(glog REQUIRED), 否则
    • op PRIVATE 依赖 glog: undefined reference to 错误, 未传递出 glog 依赖 （静态链接的可以 PRIVATE 依赖)
    • op PUBLIC 依赖 glog: target was not found 错误, 传递出 glog 依赖
  5. gcc/g++ --verbose显示编译过程
  6. strip 是一个在 Unix-like 操作系统中用于删除或剥离目标文件或对象文件的符号信息的工具。
    • 删除符号表信息：符号表包含了源代码中的函数和变量的名称。
    • 删除调试信息：debug section 被删除
  7. function tracing 只需要 symbols，不需要 debug symbols（gcc -g）。strip 之后不能跟踪
  8. 动态符号（.dynsym）vs. 局部符号（.symtab）， .symtab 包含.dynsym; 可以用命令 strip 来手动去掉局部符号表， strip 前后用readelf -s ./test看, strip 只去除了.symtab
  9. strip 之后 gdb 不能使用 function 名字加断点
  10. debug info 作用：
    • 将内存地址映射到具体某行源代码
    • 调用栈展开（stack unwinding）
  11. 在 C++中，如果函数有指定返回类型，但在实际代码中没有返回值或者在所有路径上都没有返回值，那么它的行为是未定义的（Undefined Behavior）。根据 C++标准，未定义的行为可能导致任何结果，包括程序崩溃、生成错误的结果或者在某些情况下看起来像是“正常工作”。
  12. addr2line其功能是将函数地址解析成文件名或行号的工具。 addr2line debug
  13. 查看 debug 模式：gdb l and info source
  14. -DCMAKE_BUILD_TYPE=RelWithDebInfo release 模式编译，带 debug info
  15. -DCMAKE_CXX_FLAGS="-g" cmake 编译选项添加 debug info； release 模式也适用
  16. gcc 选项顺序很重要，如果两个选项冲突，则以后一个为准。可以用-O3 -fno-inline-functions既使用-O3 的功能又关闭函数内嵌功能。

base

  1. LLVM（Low-Level Virtual Machine)
  2. llvm VS gcc
  3. 解释， 编译， 字节码(JIT)
  4. 传统的编译器通常分为三个部分，前端(frontEnd)，优化器(Optimizer)和后端(backEnd)。在编译过程中，前端主要负责词法和语法分析，将源代码转化为抽象语法树；优化器则是在前端的基础上，对得到的中间代码进行优化，使代码更加高效；后端则是将已经优化的中间代码转化为针对各自平台的机器代码
  5. llvm 和 clang 关系
  6. IR(intermediate representation): 中间语言，最常见的中间语言表述形式，是三位址码（Three address code）
  7. LLVM 是一种基础设施，是有用的编译器技术的集合，可以用于解决特定问题（例如构建 C 编译器或特效管道中的优化器）
  8. bitcode 和 bytecode 的区别是什么
  9. 注意import and export symbols
  10. echo | xt-clang -E -Wp,-v - -E 选项告诉编译器只进行预处理，-Wp,-v 选项告诉编译器打印出它在预处理阶段使用的头文件路径。
  11. 动态库全局符号覆盖的大坑 double free
    • nm libmylib1.so libmylib2.so | grep -P "^[^\\s]+ T " | cut -d' ' -f3 | sort | uniq -d | grep -v '_fini' | grep -v '_init' | c++filt 查看是否有相同全局符号
  12. Symbols 与 gcc -g 产生的 debug info 并不是一个东西
  13. function tracing 只需要 symbols，不需要 debug symbols（gcc -g）。strip 之后不能跟踪
  14. 动态符号（.dynsym）vs. 局部符号（.symtab）， 可以用命令 strip 来手动去掉局部符号表， strip 前后用readelf -s ./test看, strip 只去除了.symtab
  15. 查看依赖动态库的一些方法
    • sudo awk '/\.so/{print $6}' /proc/1/maps | sort -u 查看进行 1 链接的动态库
    • LD_DEBUG=files ./xx 查看运行时加载的库
    • readelf -d build/dog/dog_test1 | grep 'NEEDED' 查看加载用到的库
    • sudo lsof -p 1| grep mem | grep so 查看进程用到的库
